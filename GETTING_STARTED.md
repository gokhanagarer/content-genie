# Getting started

A step-by-step guide. Aim: produce a complete blog post (plan + outline + article + 3 images) in your terminal in under 10 minutes — offline, no API keys.

If anything breaks, jump to [Troubleshooting](#troubleshooting).

---

## 0. Prerequisites

You need:
- **Python 3.10+** (`python3 --version`)
- **git** (`git --version`)
- A Unix-like shell (macOS Terminal, Linux, WSL on Windows)

That's it for the demo. API keys are only needed for "real" output.

---

## 1. Clone and run the demo

```bash
git clone https://github.com/gokhanagarer/content-genie.git
cd content-genie
make demo
```

What just happened:
- `make demo` created `.venv/` and installed `requests`, `python-dotenv`, `markdown`
- Ran the full pipeline offline against the topic "Server-Side Tagging"
- Wrote artifacts to `output/<timestamp>/`

Open `output/<timestamp>/`:

```
plan.json       SEO plan (title, slug, focus keyword, audience)
outline.json    H2 section outline
article.md      complete markdown article
hero.png        placeholder hero image
info.png        placeholder infographic
photo.png       placeholder lifestyle photo
```

> **Why placeholders for images?** The offline stub writes 1×1 PNGs so the demo runs with zero network. With `LIVE=1` and a Groq key set, real LLM articles and real Pollinations.ai images replace them.

---

## 2. Switch to real LLM content

### 2.1 Get a Groq API key (free tier)

1. Sign up at https://console.groq.com — no credit card required
2. **API Keys** → **Create API Key** → copy the `gsk_...` value

### 2.2 Configure

```bash
cp .env.example .env
# open .env and set:
#   GROQ_API_KEY=gsk_...
```

### 2.3 Generate

```bash
.venv/bin/python -m src.main --topic "Your topic here" --no-wp
```

The article in `output/<timestamp>/article.md` is now a 1500–2000-word piece written by `llama-3.3-70b` instead of the offline stub.

---

## 3. Switch to real images

Set `LIVE=1` in `.env`. Pollinations.ai (free, no key) generates real images on the next run.

> Pollinations rate-limits at ~1 request / 5 s. With three images per article, expect ~20 s of generation time.

---

## 4. Publish to WordPress

### 4.1 Get an Application Password

1. WordPress admin → **Users** → your user → scroll to **Application Passwords**
2. Name it "content-genie" → **Add New Application Password**
3. Copy the password (with spaces) — you can't see it again

### 4.2 Configure

In `.env`:

```
WP_SITE_URL=https://yourblog.com
WP_USER=youradminuser
WP_APP_PASSWORD=abcd efgh ijkl mnop qrst uvwx
```

### 4.3 Publish

```bash
# Draft mode (safe — review before going live):
.venv/bin/python -m src.main --topic "Your topic" 

# Live publish:
.venv/bin/python -m src.main --topic "Your topic" --publish
```

The CLI prints the WordPress post ID and URL at the end.

---

## 5. Batch processing

Drop one topic per line in `examples/topics.txt` (or any text file):

```
Server-Side Tagging
Event Taxonomy Design
Multi-Touch Attribution
```

Then:

```bash
.venv/bin/python -m src.main --from-keywords examples/topics.txt --no-wp
```

Each topic produces its own run directory.

---

## 6. Schedule it

Run the same `python -m src.main --from-keywords ...` line from cron, Windows Task Scheduler, or a GitHub Actions cron workflow. The script is stateless — it logs every run to `output/pipeline.jsonl` so you can track what was generated when.

## Tuning the output

Everything in `.env`:

| Variable | Effect |
|---|---|
| `CONTENT_LANGUAGE=English` | Switch to `Turkish`, `German`, `Spanish`, … |
| `TARGET_WORD_COUNT=1800` | Aim for shorter or longer articles |
| `GROQ_MODEL=llama-3.3-70b-versatile` | Try `mixtral-8x7b-32768` or any other Groq-hosted model |

For deeper changes — different prompts, different LLM, different image backend — `src/plan.py` is the only file you need to read.

## Troubleshooting

| Symptom | Cause | Fix |
|---|---|---|
| `make: command not found` | `make` not installed | macOS: `xcode-select --install`. Linux: `sudo apt install build-essential` |
| Article is full of "This is a demo paragraph generated by the offline stub" | `GROQ_API_KEY` not set | Add it to `.env`, then re-run |
| 429 from Groq | Free-tier rate limit | Wait 60 s; reduce concurrent runs |
| WordPress publish: `401 Unauthorized` | Wrong user / password / not using app password | Use an **Application Password** (with spaces), not your login password |
| WordPress publish: `403 Forbidden` | User lacks `publish_posts` capability | Use an admin or editor account |
| Images all 67 bytes | `LIVE=1` not set | Add `LIVE=1` to `.env`; that triggers Pollinations instead of the placeholder |
